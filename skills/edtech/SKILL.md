# EdTech Research Domain

## Overview

EdTech research synthesizes learning science, technology platforms, and pedagogical effectiveness to understand how digital tools impact educational outcomes. This domain encompasses learning management systems (LMS), adaptive learning platforms, AI tutoring systems, and credentialing standards.

Effective research here means understanding: which LMS actually improves learning outcomes vs provides administrative convenience? How do adaptive systems personalize learning? What standards enable content interoperability?

## Key Research Dimensions

**Learning Management Systems (LMS)**
- Popular platforms: Canvas, Blackboard, Moodle, Schoology, D2L
- Feature sets: Course design, gradebook, communication, assessment
- Integration capabilities: Third-party tool ecosystem, API availability
- Mobile experience: Student and instructor mobile apps
- Accessibility: WCAG compliance, assistive technology support
- Cost models: Per-student, per-course, institutional licenses
- Institutional adoption: Which universities/schools use which platforms

**Adaptive Learning Systems**
- Personalization approach: How does system adapt to student?
- Algorithm basis: Rule-based, AI/ML, spaced repetition
- Content sequencing: How is next topic selected based on performance?
- Difficulty adjustment: How rapidly does system change difficulty?
- Learning pathways: Fixed paths vs student choice
- Effectiveness data: Research on learning outcome improvement
- Engagement metrics: Student motivation and course completion

**AI Tutoring & Intelligent Systems**
- Tutoring style: Socratic (questioning) vs direct instruction
- Domain coverage: Which subjects can system tutor effectively?
- Knowledge representation: How does system model student understanding?
- Feedback quality: Is feedback helpful or generic?
- Conversational capability: Dialogue-based vs structured interactions
- Scaling: Can system handle many students simultaneously?
- Effectiveness: Learning gains vs traditional tutoring

**Assessment & Credentialing**
- Assessment types: Multiple choice, essays, projects, competency
- Proctoring: Remote proctoring for academic integrity
- Badging & microCredentials: Digital credentials for skills
- Standards: Competency frameworks defining what students should know
- Certification: Industry-recognized credentials from educational programs
- Skills verification: Proving competency vs traditional degrees
- Labor market alignment: Do credentials employers recognize and value?

**Standards & Interoperability**
- SCORM (Shareable Content Object Reference Model): Legacy standard for content
- xAPI (Experience API): Tracking learning data across platforms
- LTI (Learning Tools Interoperability): Integrating external tools in LMS
- IMS standards: Open educational content standards
- Data portability: Can students take data between platforms?
- Accessibility standards: WCAG, ARIA for inclusive design

**Learning Science & Effectiveness**
- Cognitive load: How much information before overload?
- Spaced repetition: When and how to repeat content?
- Active learning: Are students engaged or passive?
- Retrieval practice: Testing as learning tool vs grading
- Metacognition: Student awareness of their own learning
- Transfer: Can learned skills transfer to new domains?
- Equity: Do digital tools help or hurt underserved populations?

## Analysis Frameworks

**LMS Selection Matrix** (for institution)
- **Course design**: Can instructors easily create engaging courses?
- **Student experience**: Intuitive? Mobile app quality?
- **Assessment**: Flexible assignment types and grading tools?
- **Integration**: Third-party tools, API, Zoom/Teams integration?
- **Analytics**: Can you track student learning and engagement?
- **Support**: Training, documentation, customer support quality?
- **Cost**: Per-student, institutional, pricing transparency?
- **Vendor stability**: How long in business, investment, roadmap?

**Adaptive Learning Effectiveness Assessment**
- **Learning outcomes**: What data shows learning improvements?
- **Personalization mechanism**: How much truly personalized vs surface-level?
- **Engagement impact**: Does adaptation maintain motivation?
- **Scalability**: Does system serve 10 vs 1000 students well?
- **Teacher role**: Does system enhance or replace instructors?
- **Cost-benefit**: Time saved vs effectiveness gain?
- **Implementation burden**: Teacher training and setup effort?

**AI Tutoring Viability Framework**
- **Domain suitability**: Math/STEM vs humanities/open-ended learning?
- **Student level**: Elementary, high school, undergraduate, professional?
- **Scale**: One-on-one supplement vs primary instruction?
- **Effectiveness data**: Published learning outcome improvements?
- **Cost**: Per-student cost vs human tutoring?
- **Equity**: Does tutoring help struggling students most?
- **Teacher acceptance**: Do educators see as tool or replacement?

**Credentialing & Skills Alignment**
- **Employer recognition**: Do hiring companies value credential?
- **Skill currency**: Does credential align with current job market?
- **Portability**: Can credential transfer between platforms/institutions?
- **Verification**: Can employers easily verify claimed skills?
- **Update speed**: How quickly updated when skills evolve?
- **Completeness**: Does credential demonstrate comprehensive competency?
- **Cost**: Student cost vs perceived value in labor market?

**EdTech Accessibility Audit**
- **WCAG 2.1 compliance**: Minimum AA level required
- **Assistive tech support**: Screen readers, voice control, etc.
- **Content accessibility**: Videos captioned, PDFs tagged
- **Universal Design**: Built for all learners, not added later
- **Evaluation**: Third-party accessibility audit results
- **Remediation**: Plan to address identified gaps
- **Ongoing**: Accessibility maintenance and updates

## Key Metrics to Always Collect

For EdTech decisions, track:
1. **Learning outcomes**: Test scores, skill assessment, competency mastery
2. **Engagement**: Course completion rate, time on platform, student satisfaction
3. **Accessibility**: WCAG compliance level, accessibility incident count
4. **Adoption**: Instructor usage rate, student active participation
5. **Cost**: Per-student, per-course, total implementation cost
6. **Equity**: Outcome gaps by demographic group
7. **Credential value**: Employer recognition, job placement impact

## Computational Workflows

**Pre-built Analysis Scripts**
- `learning_outcome_analyzer.py` — Compare student achievement across platforms using statistical hypothesis testing with scipy
- `lms_cost_calculator.py` — Compute total cost of ownership per student (licensing + implementation + support) using pandas
- `engagement_metric_aggregator.py` — Calculate course completion rates, time on platform, and engagement scores
- `accessibility_compliance_scorer.py` — Audit WCAG compliance levels and forecast remediation effort with scoring

**Dynamic Computation Examples**
- When selecting LMS, compute per-student cost over contract term accounting for growth and batch sizes
- When evaluating adaptive learning, run statistical comparison of outcome improvements across student cohorts
- When assessing engagement, generate confidence intervals on completion rate changes between platform versions

**Output Artifacts**
- Learning outcome comparison table (platform x metric with effect sizes and p-values)
- Cost per student breakdown (licensing/implementation/support over contract term, amortized)

- Mermaid flowchart showing LMS selection criteria and decision path

## Common Pitfalls to Avoid

- **LMS hype**: Flashy features don't ensure learning; focus on pedagogy
- **Cost surprise**: "Cheap" LMS costs more in implementation and support
- **Data security neglect**: Student data privacy crucial (FERPA, GDPR)
- **Mobile afterthought**: Platform desktop-first when students mobile-first
- **Accessibility failure**: Non-compliant platform excludes students, risks litigation
- **Engagement metrics confusion**: Time on platform ≠ learning (busy work looks good)
- **AI tutor over-promise**: System works on specific math topics, fails on open-ended
- **Credential skepticism**: Employer unrecognized credentials waste student time
- **Integration gaps**: Disconnected systems create poor user experience
- **Teacher resistance**: Technology without professional development fails adoption
